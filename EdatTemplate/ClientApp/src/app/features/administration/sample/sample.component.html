The EDAT Template Application is an Single Page Application (SPA) architecture based on .NET Core 3.1.100 (server app) and Angular 8.2.14 (client app).

The following components demonstrate common-use scenarios encountered in application development. The samples are all located in /ClientApp/src/app/features/administration/sample and can be pruned out once they are no longer needed. This template application does contain many useful core components in /ClientApp/src/app/components and services in /ClientApp/src/app/services that should remain as part of your application.

Sample Component/Store Service Source Location: ClientApp/src/app/features/administration/sample/
Common Component Source Location: ClientApp/src/app/components/common/
Data Service Source Location: ClientApp/src/app/services/data/

Sample Charts Component
Sample Component: sample-charts.component.(ts|html)
Common Component: chart-to-table.component.(ts|html)
This component uses the ngx-charts library. This example demonstrates how to make an API call to get data to bind to a chart. The ngx-charts library is a client-side Angular charting framework that draws SVG charts that are reactive to data binding updates. The charts use aria- attributes for accessibility compliance.

Sample Data Component
Sample Component: sample-data.component.(ts|html) - Data Grid with Sort, Filter, and Paging
Sample Component: sample-modal.component.(ts|html) - Modal Add and Update
Sample Store Service: sample-store.service.ts - Data Management Observable Pattern Service
Common Components: filter-field.component.(ts|html), sort-button.component.(ts|html), staff-picker.component.(ts|html)
Data Services: data-navigation.service.ts, excel-export.service.ts, staff.service.ts
This component demonstrates the use of a store service to manage client-side application state. The data grid supports sorting, filtering, and paging, and is composed of smaller components that implement these features. Adding, updating, and deleting data is implemented via modal dialogs, and all validations are performed server-side. This component also demonstrates exporting the filtered data to Excel, and using moment.js to calculate the age based on the birth date client-side.

Sample Email Component
Sample Component: sample-email.component.(ts|html)
Common Component: staff-picker.component.(ts|html)
Data Service: email.service.ts
API: EmailController.cs
Infrastructure Service: EmailService.cs
This component demonstrates the use of the email service to send an email to a selected user using the SendGrid API. The user is selected with the Staff Picker component that uses the Azure API Management FDOT core services (Arculus) server-side.

Even though the sample uses the Angular email service to demonstrate sending an email, the recommended approach is to only send emails from server controllers using the .NET email service.

Sample Documents Component
Sample Component: sample-documents.component.(ts|html)
Common Component: file-upload.component.(ts|html)
Data Service: blob.service.ts
API: StorageController.cs
Infrastructure Service: BlobStorageProvider.cs
This component demonstrates the use of the file-upload component and BLOB storage service to store documents and files. The BLOB storage service uses Azure BLOB Storage server-side. You will need the Azure Storage Emulator for local development.

Sample EDMS Component
Sample Component: sample-edms.component.(ts|html)
Common Component: edms-file-upload.component.(ts|html)
Data Service: edms.service.ts
API: EdmsController.cs
Infrastructure Service: EdmsService.cs
This component demonstrates the use of the edms-file-upload component and supporting services to interface with the FDOT EDMS system for the storage and retrieval of enterprise documents. All EDMS servers (Central Office and District) are supported. The EDMS service uses the Arculus EDMS API. The template application implementation provides the following: creating a new document from an uploaded file, adding a new version to an existing EDMS document from an uploaded file, and downloading an existing document in the EDMS system.

Sample PDF Document Generation
Sample Component: sample-pdf-document.component.(ts|html)
Data Service: pdf-document.service.ts
This component demonstrates a service that uses the jsPDF library to create a PDF document from an HTML template. The HTML is separated into multiple pages and contains various elements like text, a hyperlink, a table, and an image. An input is bound to the document title to demonstrate dynamic content binding. This is just to demonstrate one way to use the jsPDF library to create PDF documents, but the library is capable of much more. This example was meant to show how a letter might be generated in the application.

Sample Signature Component
Sample Component: sample-signature.component.(ts|html)
Common Component: signature-field.component.(ts|html)
API: SignatureController.cs
Infrastructure Service: BlobStorageProvider.cs
This component demonstrates the signature-field component which captures a signature drawing on a canvas. It uses the signature_pad library for the drawing implementation and BLOB storage as a data store. It uses a BLOB ID (string) as the model, and will load a previously saved signature (as an image) and save a new signature drawing to a new BLOB. It also supports a read-only mode.

Sample Map Component
Sample Component: sample-map.component.(ts|html)
Common Component: map-field.component.(ts|html)
API: MapController.cs
Infrastructure Service: BlobStorageProvider.cs
This component demonstrates the map-field component which captures and displays geo-spatial data. The component uses the leaflet library and plugins for the map and BLOB storage as a data store. It uses a BLOB ID (string) as the model, and will load one or more previously saved feature collections to the map and save a new feature collection of features currently on the map to a new BLOB. It also supports a read-only mode.

The component map supports the following: drawing (line, polygon, square) features, editing feature styles (line color, line style, fill color, and description), deleting features, searching feature descriptions, and ESRI address search.
